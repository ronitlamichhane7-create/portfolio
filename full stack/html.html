Section Two

HTML builds the structure.

HTML gives meaning to content and forms the skeleton of every website. It is the first layer of front-end development.

Semantic Markup

Good HTML uses meaningful elements such as header, nav, main, section, article, and footer.

Explanation

HTML stands for HyperText Markup Language. It is the standard language used to create the structure and content of a web page. HTML does not focus on visual design or complex behaviour; instead, it gives meaning and organisation to the information shown in the browser. Every website uses HTML in some way because it defines the elements that appear on the page, such as headings, paragraphs, images, links, lists, tables, forms, sections, and navigation areas. A strong understanding of HTML is important because it forms the foundation of every front-end project. Without good HTML structure, CSS styling and JavaScript functionality become harder to manage.

Modern HTML should be semantic, accessible, and well organised. Semantic HTML means using elements that describe the purpose of the content. For example, a header should be placed inside a header element, navigation links should be placed inside a nav element, and the main page content should be placed inside a main element. This helps browsers, search engines, screen readers, and developers understand the page more clearly. Accessibility is also important because websites should be usable by people with different needs. Good use of alt text for images, clear headings, labelled forms, and readable content improves the user experience. In this portfolio, the HTML demonstrates a professional page structure with a fixed navigation bar, hero area, content sections, cards, buttons, images, and footer. The layout is divided into meaningful sections so that the code is easier to read, maintain, and improve.

HTML best practices

Use semantic elements, provide accessible image alt text, place headings in order, and group related content in sections or articles. This helps search engines, screen readers, and future developers understand the page.

A clean HTML structure also makes it easier to add styling and scripts later. For example, a form can be enhanced with CSS for layout, and JavaScript can add validation or interactive feedback without rewriting the entire page.
